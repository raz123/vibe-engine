### Architecture Decisions (MVP)

- **Backend**: Python + FastAPI
- **Database**: PostgreSQL (Phase 1 local, Phase 2+ cloud)
- **Cache**: Redis (optional, for queues/cache)
- **GPU**: CUDA-capable for Demucs/ML inference

### Audio Pipeline
- yt-dlp + FFmpeg for extraction
- librosa + Essentia for analysis (BPM, key, structure, energy)
- Demucs for stem separation
- Rubber Band Library + FFmpeg for real-time DSP

### Transition Engine (heart of system)
6 transition types: Simple Crossfade, EQ Blend, Drum Carry, Vocal Swap, Drop Transition, Echo/Reverb Exit
Hard constraints: no key clashes, no vocal overlap, no off-beat cuts, no mid-phrase cuts

### Recommendation Engine
Track graph model with energy/mood/BPM/key/genre embeddings
Session phases: warmup -> buildup -> peak -> sustained high -> cooldown
Feedback loop: reinforcement scoring on skips/likes/duration

### Vibe Modes
Club, Chill, Focus, Late Night, Aggressive, Euphoric, Deep House, Experimental

### Milestones
1. Basic playback + crossfade
2. BPM + key-aware transitions
3. Phrase-aware transitions
4. Stem-separated smart transitions
5. AI emotional queue planning

### Legal
- No permanent hosting of copyrighted music
- No redistribution
- Temporary local caching only (LRU, auto-delete, configurable cap)
- User-supplied playback sources only
- All processing client-side or ephemeral

### Design Philosophy
Not a playlist player — an autonomous vibe engine.
Transition quality > UI. Song selection > transition effects.
Avoid dead air. Avoid vocal collisions. Prioritize emotional continuity.
